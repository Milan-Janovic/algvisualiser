We will use integer representation further on, but operations like XORs or shifts are done in bitwise manner.
Firstly, we split key K into left KL and right KR part.
KL = {{keyL}}
KR = {{keyR}}
Next, we put KL throught F-function and store result as Kr.
Kr = F-function({{keyL}}) = {{keyTemps.0}}
Then we XOR Kr with KR and store result as Kr.
Kr = Kr XOR {{keyR}} = {{keyTempsXORed.0}}
Now we swap KL and KR, which means we set KR equal to KL and KL equal to Kr.
KL = Kr = {{keyLs.0}}
KR = KL = {{keyRs.0}}
We need to repeat the process 3 more times.
So we split key K into left KL and right KR part.
KL = {{keyLs.0}}
KR = {{keyRs.0}}
Put KL throught F-function and store result as Kr.
Kr = F-function({{keyLs.0}}) = {{keyTemps.1}}
XOR Kr with KR and store result as Kr.
Kr = Kr XOR {{keyRs.0}} = {{keyTempsXORed.1}}
Swap KL and KR.
KL = Kr = {{keyLs.1}}
KR = KL = {{keyRs.0}}
Repeat the process 2 more times.
Split key K into left KL and right KR part.
KL = {{keyLs.1}}
KR = {{keyRs.1}}
Put KL throught F-function and store result as Kr.
Kr = F-function({{keyLs.1}}) = {{keyTemps.2}}
XOR Kr with KR and store result as Kr.
Kr = Kr XOR {{keyRs.1}} = {{keyTempsXORed.2}}
Swap KL and KR.
KL = Kr = {{keyLs.2}}
KR = KL = {{keyRs.2}}
Repeat the process 1 more time.
Split key K into left KL and right KR part.
KL = {{keyLs.2}}
KR = {{keyRs.2}}
Put KL throught F-function and store result as Kr.
Kr = F-function({{keyLs.2}}) = {{keyTemps.3}}
XOR Kr with KR and store result as Kr.
Kr = Kr XOR {{keyRs.2}} = {{keyTempsXORed.3}}
Concatenate KL and KR to get Ka.
Ka = (KL || KR) = {{Ka}}

K subkeys are generated generated from right, left parts of keys K, Ka and circular rotations of these parts by 15, 60, 94 and 111 bits to the left.
First we split plaintext PT into left PTL and right PTR part.
Then we XOR PTL with kw0 and PTR with kw1 and we get PTL1 nad PTR1 .
PT = {{PT}}
Next we put PTL throught F-function with kl1-6 subkeys, XOR the result with PTR1-6 and store the result as PTr1-6.
Then we swap PTL1 and PTR1, same way we did in case of Ka key generation.
We repeat this process 6 times and then we put PTL6 through FL-function and PTR6 through
FL-1-function. As result we get final PTL6 and PTR6

Generally, decryption process is encryption process in reverse manner.
First we split cipher text CT into left CTL and right CTR part.
Then we XOR PTL with kw3 and PTR with kw4.
CT = {{CT_final}}
Next we repeat the encryption process in reverse order so we put PTL throught F-function 18 times with kl18-1 subkeys, each time we XOR the result of F-function with PTR18-1.
Then we swap PTL and PTR, same way we did in case of Ka key generation.
Remember that after every six iterations of putting PTL throught F-function, XOR-ing the result with PTR and swaping PTL and PTR we put PTL through FL-function and PTR through FL-1-function.
